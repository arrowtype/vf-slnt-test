Browser Test: font-style: oblique XXdeg & variable font Slant axis

Expected behavior

Per the CSS Spec: Fonts Module Level 4, font-style, variable fonts with a Slant axis with negative values should be possible to control via CSS with a positive value of oblique. That is, CSS such as font-style: oblique 10deg; should result in clockwise-slanting text when the font has a slnt axis with values 0 to -10.

Test: positive oblique in @font-face

@font-face {
  font-family: 'Inter1';
  font-weight: 100 900;
  font-style: oblique 0deg 10deg; /* follows CSS spec */
  src: url('fonts/Inter.var.ttf');
}

font-variation-settings: 'slnt' -10;

font-style: oblique 10deg;

font-style: oblique -10deg;

font-style: italic;

<i>

Test: negative oblique in @font-face

@font-face {
  font-family: 'Inter2';
  font-weight: 100 900;
  font-style: oblique 0deg -10deg; /* testing a different way */
  src: url('fonts/Inter.var.ttf');
}

font-variation-settings: 'slnt' -10;

font-style: oblique 10deg;

font-style: oblique -10deg;

font-style: italic;

<i>

Results as of Jan 27, 2020

All tested browsers support font-variation-settings: 'slnt' -10, but support for font-style: oblique XXdeg is much worse:
  • Chrome (81.0.4040.0): No support.
  • Safari (13.2, WebKit 15610.1.1): No support.
  • Firefox (72.0a1, 72.0.2): closest to support, but only shows clockwise slant for a *negative* oblique value (opposite of what should happen, per CSS spec)

Test source: https://github.com/arrowtype/vf-slnt-test
