Browser Test: font-style for variable font with Slant axis

Expected behavior

Variable fonts with negative, clockwise-sloping Slant axis values should be possible to control via CSS with a positive value of oblique (CSS Spec, font-style). For example, CSS such as font-style: oblique 10deg; should result in clockwise-slanting text when the font has a slnt axis with values 0 to -10 (as required by the OpenType spec: “a typical, right-leaning oblique design will have a negative slant value”).

Additionally, font-style:italic should activate the Slant axis if an Italic axis is not present. Per the CSS Spec: “Italic: Matches against a font that is labeled as an italic face, or an oblique face if one does not exist ...For TrueType / OpenType fonts that use variations, the slnt variation is used to implement oblique values”

Test: positive oblique in @font-face

@font-face {
  font-family: 'Inter1';
  font-weight: 100 900;
  font-style: oblique 0deg 10deg; /* follows CSS spec */
  src: url('fonts/Inter.var.ttf');
}

All following elements should be forward-slanted (clockwise-sloped), except for font-style: oblique -10deg;

font-variation-settings: 'slnt' -10;

font-style: oblique;

font-style: oblique 10deg;

font-style: italic;

<i>

font-style: oblique -10deg;

Test: negative oblique in @font-face

@font-face {
  font-family: 'Inter2';
  font-weight: 100 900;
  font-style: oblique 0deg -10deg; /* testing a different way */
  src: url('fonts/Inter.var.ttf');
}

The above code should only apply to a “backslanted” font with a counter-clockwise Slant axis of 0 to positive 10. In a font with a clockwise-slant, the font-style declaration above is incorrect.

font-variation-settings: 'slnt' -10;

font-style: oblique;

font-style: oblique 10deg;

font-style: italic;

<i>

font-style: oblique -10deg;

Results as of Jan 27, 2020

All tested browsers support font-variation-settings: 'slnt' -10, but support for font-style: oblique XXdeg is sparse. As tested on macOS, Windows, iOS, and Android:
  • Chrome (81.0.4040.0): No support. (Bug tracked at #859869)
  • Android 9.0 Chrome: No support.
  • Edge (79 Beta): No support.
  • Opera (67): No support.
  • Safari (13.2, WebKit 15610.1.1): No support. Applies faux-slant of 20deg.
  • Safari iOS 13: No support. Applies faux-slant of 20deg.
  • Firefox (72.0a1, 72.0.2): Closest to support, but only shows clockwise slant for a *negative* oblique value (opposite of what should happen, per CSS spec).

Note: both Chrome and FireFox appear to work well if a variable font has a clockwise slant with a *positive* value; however, fonts using this approach contradict the OpenType spec and therefore do not provide a valid test case.
